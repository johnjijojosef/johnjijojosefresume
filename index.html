5) Secure Headers (Apply protections)

This mock scan shows missing security headers. Click “Apply Headers” to harden the app and reveal the flag.

Report
  • Missing: Content-Security-Policy
  • Missing: X-Content-Type-Options
  • Missing: Strict-Transport-Security

These headers mitigate clickjacking, MIME sniffing, downgrade attacks, and inline script risks.

6) Least Privilege (Pick the safest policy)

Which policy grants read-only access to only the reports-2026 bucket objects?

Candidates

A (Overly broad)

{
  "Effect": "Allow",
  "Action": "s3:*",
  "Resource": "*"
}

B (Least privilege)

{
  "Effect": "Allow",
  "Action": ["s3:GetObject"],
  "Resource": "arn:aws:s3:::reports-2026/*"
}

C (Too broad on actions)

{
  "Effect": "Allow",
  "Action": ["s3:*"],
  "Resource": "arn:aws:s3:::reports-2026/*"
}
Prefer the smallest action set and the narrowest resource scope needed to perform the job.
🔐 FLAG{LEAST_PRIVILEGE}
