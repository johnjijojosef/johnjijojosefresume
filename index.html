6) SIEM Tuning (Sequence: fail → success)

Choose the query that detects ≥3 failed logins followed by a success from the same IP within 5 minutes.

A

SigninLogs
| where ResultType != 0

B

SigninLogs
| where ResultType != 0
| summarize fails=count() by IPAddress, bin(TimeGenerated, 5m)
| join kind=inner (
    SigninLogs
    | where ResultType == 0
    | project IPAddress, TimeGenerated
) on IPAddress
| where fails >= 3 and TimeGenerated1
  between (TimeGenerated .. TimeGenerated + 5m)

C

SigninLogs
| where ResultType == 0
| summarize successes=count()
Think about grouping failures by IP in a short time window, then confirming a success for that IP in the same window.
